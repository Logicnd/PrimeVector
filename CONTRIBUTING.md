# Contributing to PrimeVector

We welcome contributions from the community! To ensure a smooth experience, please follow these guidelines.

## Code of Conduct

All participants are expected to adhere to our [Code of Conduct](CODE_OF_CONDUCT.md). Please read it before contributing.

## Getting Started

1. Fork the repository and clone your fork locally.
2. Install the dependencies by running `pip install -r requirements.txt` and set up pre‑commit hooks if configured.
3. Create a new branch for your changes (e.g. `git checkout -b feature/my‑feature`).
4. Make your changes and write appropriate tests.
5. Run the test suite and linters to ensure everything passes.
6. Commit your changes with clear, descriptive messages.
7. Push your branch and open a pull request.

## Commit Messages

Use clear commit messages that describe your changes. Following the conventional commit format (e.g. `feat: add new module`) is encouraged but not required.

## Pull Requests

When opening a pull request:

- Describe what your change does and why it is needed.
- Link related issues if applicable.
- Ensure all checks pass (tests, linters, etc.).
- Be responsive to feedback during the review process.

## Issues

If you find a bug or have a feature request, please open an issue with a descriptive title and provide context. Include reproduction steps and expected behavior for bugs.

## Documentation

If your change involves modifying or adding functionality, update the documentation accordingly.

Thank you for helping make PrimeVector better!
